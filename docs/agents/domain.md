# Domain Docs

This is a single-context repo. Read `CONTEXT.md` at the repo root and relevant ADRs under `docs/adr/` before exploring the codebase.

If these files do not exist, proceed silently. They are created lazily as domain language and architectural decisions are resolved.

Use the glossary vocabulary from `CONTEXT.md` when naming domain concepts. Surface any conflict with an existing ADR explicitly.
